Kerno Platform Guide

Internal platform for CaixaResearch Core Facilities — Platforms team

Getting started

Logging in

Navigate to index.html. Enter your username and password. After a successful login:

  • Admins are redirected to the Dashboard.
  • All other users are redirected to the Kanban board.
Your session lasts 8 hours. If the page redirects you to login with an "expired" message, just sign in again — your work is saved in the database.

Navigation

The top-right corner of each page shows your username and a sign-out link. Use the browser's address bar or any in-page link to move between pages:

  • dashboard.html — Stats and overview
  • kanban.html — Project board
  • booking.html — Resource bookings
  • requests.html — Service requests

Roles

Every user has one role that controls what they can see and do across the platform.

Role Who Capabilities
admin Platform managers Full access. Manage groups, approve bookings, review requests, see all projects, access Users & Groups panel.
leader Group scientists / PIs See all projects and bookings for their group. Can query the AI with group context.
manager Lab operations Same as leader — group-wide visibility and AI group queries.
user Regular members Own projects + projects where they are @mentioned. Own bookings and requests.

Booking access is additionally controlled by a per-user bookings permission. An admin must grant this before a user can make bookings.

Dashboard

The dashboard (dashboard.html) provides an at-a-glance overview of platform activity. It is accessible to admin users.

Summary cards

  • Active projects — total non-archived projects across all groups.
  • Team members — count of all registered users.
  • Total requests — all service requests ever submitted.
  • Recent requests — requests submitted in the last 7 days.

In-progress & blocked tasks

Two panels highlight tasks that need attention right now: tasks currently In Progress and tasks marked as Blocked. Each entry links back to the project it belongs to.

Financial report

The financial section shows all billed task cards that have store order items and were not excluded from the report. For each entry you can see:

  • Project name and task title
  • Line items from the store order
  • EUR totals

A card appears here only when it has been through the Bill → flow on the Kanban board (see Billing a card) and the "Include in financial report" toggle was left on.

Team panel & audit log

The Team panel lists all members with their roles and group memberships. The Audit log shows a chronological record of all actions (project edits, booking changes, request reviews, etc.).

Kanban board

The Kanban board (kanban.html) is where day-to-day experiment tracking happens. Projects live here as cards organised into task columns.

Scope views

Use the scope selector at the top of the left sidebar to filter which projects are shown:

  • Mine — projects you own.
  • Mentioned — projects where any task note contains @yourusername.
  • Group — all projects from members of your group. Available to leader, manager, and admin.

Creating a project

  1. Click + New project at the top of the sidebar.
  2. Enter a project name and confirm.
  3. The project opens immediately and is ready for tasks.

Cards & tasks

Each project has a board with four columns:

  • To Do — not started yet.
  • In Progress — actively being worked on.
  • Blocked — waiting on something external.
  • Done — completed.

Adding a task

Click + Add task at the bottom of any column. Enter a title and press Enter.

Editing a task

Click any task card to expand it. You can edit:

  • Title — click the title text to edit inline.
  • Notes — rich-text notes with a Markdown toolbar (see below). Supports @username mentions — type @ to get a suggestion list.
  • Assignee — pick a member from the dropdown.
  • Column — use the column selector or drag (if supported).
  • Store order — add billable line items (product name, quantity, unit price). The card shows a total in EUR.

Markdown notes

The notes field has a formatting toolbar. Supported syntax:

  • **bold** → bold
  • *italic* → italic
  • ~~strikethrough~~ → strikethrough
  • ==highlight== → highlighted text
  • `code` → inline code
  • - item → bullet list
  • 1. item → numbered list
  • # Heading → heading
Click a toolbar button while text is selected to wrap it. Click without a selection to insert a placeholder. For lists, place your cursor on a line and click the list button to add/remove the prefix.

Moving tasks

Use the column selector inside the expanded card view. Changes are saved automatically.

Deleting a task

Open the task card and click the trash icon. You will be asked to confirm.

Billing a card

When an experiment is complete and has billable items, you can finalise it with the Bill → button. This button appears only on task cards that have at least one store order line item.

How to bill a task

  1. Open the task card and confirm store order items are filled in.
  2. Click Bill → (visible on the card in the collapsed board view).
  3. A confirmation modal appears showing the task title and total.
  4. The "Include in financial report" toggle is on by default. Leave it on if this expense should appear in the Dashboard financial report. Turn it off if the billing was an error or should be excluded for any reason.
  5. Click Bill → to confirm.

After billing

The task moves out of the active board columns and appears in a Billed section below the board. It shows the task title, EUR total, and (if excluded) an "excl. report" badge.

Billed tasks are not visible in the AI assistant context. If you need the AI to reason about a task, reopen it first.

Reopening a billed task

Hover over a billed task and click ↩ reopen. The task returns to the Done column and is no longer flagged as billed.

Financial report

Billed tasks with "Include in financial report" turned on appear in the Dashboard financial section. Tasks excluded from the report do not appear there.

AI assistant

Click AI in the right-side panel tabs to open the AI assistant. It is powered by Claude (Anthropic).

Context

When a project is open on the board, the AI automatically receives the full task list for that project (excluding billed tasks) as context. You can ask questions like:

  • "Summarise what's left to do."
  • "Which tasks are blocked and why?"
  • "Draft an update email for this project."
  • "What was the last note added to the sequencing task?"

If no project is selected, the AI sees a compact list of all projects in the current scope (names and task counts only) and you can ask broader questions.

Scope-aware queries

leader / manager users with the Group scope active can ask about all group projects at once when no single project is selected.

Closing a project

When a project is fully done, you can close (archive) it to remove it from the active sidebar list.

  1. Open the project on the board.
  2. Click the archive link next to the project title.
  3. Confirm in the modal.

Archived projects disappear from the default view. Toggle Show archived in the sidebar to view or restore them.

Closing a project has no financial effect. Billing is per task card — use the Bill → flow before closing if any tasks need to go to the financial report.

Bookings

The Bookings page (booking.html) shows a weekly calendar grid for reserving shared lab resources. You need the bookings permission to make reservations — contact an admin if you do not have access.

Making a booking

  1. Select a resource from the left panel.
  2. Navigate to the desired week using the arrow buttons.
  3. Click a time slot (or a day for full-day resources) to open the booking modal.
  4. Fill in any required custom fields for that resource.
  5. Click Book. If the resource requires approval, the booking is created with pending status.

Booking statuses

  • Pending — submitted, waiting for admin approval.
  • Approved — confirmed, slot is reserved.
  • Cancelled — cancelled by the user.
  • Rejected — declined by an admin.

Overlap prevention

The platform prevents double-bookings at two levels — a pre-check in the server and a database constraint. If a slot is already taken, you will see an OVERLAP error inline in the modal.

Admin — managing resources

admin users see an Admin panel on the left. From there you can:

  • Add, edit, or deactivate resources.
  • Set booking type: time slot, full day, or both.
  • Toggle approval requirement per resource.
  • Add custom fields (text, dropdown, multiselect) that appear in the booking form.
  • Approve or reject pending bookings.

Service requests

The Requests page (requests.html) lets users submit structured service requests (e.g. Genomics, Cell Sorting, or other configured types).

Submitting a request

  1. Choose a request type from the tabs.
  2. Fill in the form fields (configured per type by admins).
  3. Click Submit. You will see your request in the list with pending status.

What happens after submission

  • The appropriate reviewer (based on request type) is notified.
  • On approval: a new Kanban project is automatically created with all your request details as a single "To Do" task. You also receive a confirmation email.
  • On rejection: you receive an email with the reviewer's notes.

Admin — reviewing requests

admin users see an Admin review section. Click any pending request to expand it, then choose Approve or Reject and add optional notes.

Admin — configuring request types

Request type definitions (label, approver role, form fields) are configurable from the admin panel. You can add or remove field types: text, dropdown, and multiselect.

Admin tasks

Managing users & groups

Open the Users & Groups panel on the Kanban board (visible to admin only). From here you can:

  • View all members and their roles.
  • Create and delete groups.
  • Assign members to groups with a role (member / leader / manager).
  • Grant or revoke the bookings permission for individual users.

Weekly report

Every Monday at 8:00 UTC, the platform automatically sends a weekly email to all members with an email address. The report includes:

  • All currently blocked tasks (with notes).
  • A summary of actions logged in the previous week.

The report is generated by Claude and sent via Brevo.

Audit log

All significant actions (project edits, booking changes, request reviews, admin changes) are recorded in the audit log visible on the Dashboard.

Tips & shortcuts

Mentions

In any task notes field, type @ followed by a username to mention a team member. Mentioned tasks appear in that user's Mentioned scope on the Kanban board.

Store orders

Use the store order panel inside a task card to record reagents, consumables, or any billable items. Enter a product name, quantity, and unit price — the total updates automatically. Once items are entered, the Bill → button becomes available on the card.

Finding archived projects

Toggle Show archived at the bottom of the project sidebar. Archived projects appear in a separate list and can be restored at any time.

Session expiry

Sessions last 8 hours. If you see the login page unexpectedly, your session expired — sign in again. All data is in the database and will be waiting for you.

Booking conflicts

If you see an OVERLAP error when booking, the slot was taken between when you loaded the page and when you submitted. Refresh the calendar and pick another time.

Pro tip: Use the AI assistant with a project open to quickly get a status summary before a meeting. Ask "What's the status of this project?" and get a structured overview of all tasks and their current column.
