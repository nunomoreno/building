Getting Started

Kerno is the internal platform for CaixaResearch Core Facilities. It combines project tracking, resource bookings, service requests, and financial reporting in a single interface hosted on GitHub Pages.

Logging in

Open index.html and enter your username and password. Sessions last 8 hours — after that you will be redirected to the login page automatically. After login, admins land on dashboard.html; all other users land on booking.html.

Roles

Role Capabilities
Admin Full access. Manages groups, approves bookings, reviews requests, manages grants and users.
Leader Group scientist. Sees all projects, bookings, and requests within the group. AI queries with group scope.
Manager Lab operations. Same access as Leader — sees all group projects and bookings.
User Standard member. Sees own projects and @mentioned projects. Manages own bookings and requests.
ℹ

Booking access is controlled per-user by an explicit bookings permission on top of the role. If the Bookings page is inaccessible, ask an admin to enable it for your account.

Dashboard

The dashboard (dashboard.html) is your command centre. It surfaces platform-wide metrics and provides quick navigation to all modules.

Overview stats

Three summary cards appear at the top of the page:

—
Active projects
—
Team members

Below the stats you will find a Recent Activity feed of the latest actions across the platform, and a Team panel listing all group members with their roles.

Expenses & CSV export

The Resource Usage Costs table aggregates spending from two sources:

  • Approved resource bookings that carry a cost
  • Billed store orders from Kanban task cards

Use the year selector and scope toggle (your own costs / group) to filter the table. Click ⬇ Download CSV to export the current view as a comma-separated file.

⚠

Only bookings with approved status and tasks that have been explicitly billed appear in the expense table. Pending, cancelled, or unbilled entries are excluded.

Kanban Board

The Kanban board (kanban.html) is the core project management tool. Each project has four columns: To Do, In Progress, Blocked, and Done.

Scope views

The scope selector filters which projects appear in the left sidebar:

Scope Shows Available to
Mine Projects you own Everyone
Mentioned Projects where any task note contains @yourname Everyone
Group All projects from every member of your group Leader & Manager

Creating & managing projects

Click + New project in the left project sidebar to create a project. Give it a name and it will appear as an empty board with four columns.

To close a project, hover over its name in the sidebar — a close icon appears on the right side. Closed projects are archived and no longer shown in the active list, but their data is preserved.

💡

Projects created from approved service requests appear prefixed with the request type, e.g. [Genomics] Sample batch 12. They arrive with a single To Do task pre-filled with all submission metadata.

Task cards

Each card contains:

  • Title — click to rename inline
  • Notes — click to open the markdown editor
  • Assignee — optional team member assigned to the task
  • Move arrows — ← and → shift the card to the adjacent column

Add a new card by clicking + Add task at the bottom of any column.

Store orders

Task cards can hold store order items — consumables or reagents ordered for the task. To add an item:

  1. Open a task card and find the Store Orders section
  2. Enter a SKU (product code) and quantity
  3. Optionally assign a grant from the grants you are a member of
  4. Click Add item

Multiple SKU items can be added to the same task. Each item shows its SKU, quantity, grant code, and unit cost if defined in the product catalogue.

Billing a task

Once store order items are attached to a task, a Bill → button appears on the card. Clicking it opens the billing modal showing an itemised cost summary.

The modal includes a toggle labelled Include in financial report, which is on by default. Leave it enabled so the cost appears in the Dashboard expense table and CSV export.

Confirming billing moves the task to the Billed section below the board. The card is no longer editable from within the columns.

Billed section

Below the four Kanban columns sits a compact Billed list. Each entry shows the task name, SKU items, and cost. The section header displays the running EUR total for all billed tasks in the current project.

Hover over a billed task to reveal an ↩ Reopen button. This returns the task to the Done column so you can correct errors before re-billing.

AI Assistant panel

The AI Assistant panel opens from the right side of the board. It is scoped to the project currently open — it can see all tasks, notes, and columns for that project.

The assistant can answer questions and take actions:

  • Add new tasks to a specified column
  • Move tasks between columns
  • Update task titles or notes
  • Delete tasks
  • Summarise what is blocked or in progress
💡

The AI only sees the currently open project. Switch projects in the sidebar before asking about a different project.

Audit Log panel

The Audit Log panel records every action on the platform: project creation, task moves, billing events, booking changes, request approvals, and more. Each entry shows the username, action type, details, and timestamp. Admins also have access to a Users & Groups panel from the same right-side drawer.

Bookings

The Bookings page (booking.html) lets you reserve lab equipment, instruments, and other shared resources. Access requires the bookings permission on your account.

Calendar grid

The page displays a weekly grid — columns are days, rows are time slots. Use the arrow buttons in the header to navigate between weeks.

To create a booking:

  1. Select a resource from the left panel
  2. Click an empty cell on the grid for your desired day and time
  3. Fill in start time, end time (or full day), notes, and any custom fields for the resource
  4. Optionally assign a grant for cost tracking
  5. Click Submit

Resources support three booking modes:

  • Timeslot — pick specific start and end times
  • Full day — blocks the entire day
  • Both — a toggle in the form lets you choose per booking
🚫

Overlapping bookings are blocked at two levels: a server pre-check and a database exclusion constraint. If another user books the same slot while you are filling in the form, you will see an OVERLAP error inline in the modal.

Approval workflow

Resources that require admin approval show bookings in Pending status until reviewed. The flow:

Pending → Approved or Rejected or Cancelled or Released

Resources without approval required move directly to Approved on submission. Admins manage pending bookings from the admin panel on the right side of the bookings page.

Delete tolerance & Release

Admins can configure a Delete tolerance (in hours) per resource. This prevents last-minute cancellations that leave slots empty with no time for others to book.

  • Before the tolerance window — the Delete button is shown. Deleting removes the booking completely.
  • Inside the tolerance window (within X hours of the booking start) — the Delete button is replaced by an amber Release button.

Release — the booking is hidden from the calendar so the slot appears free and others can book it, but the entry remains in the financial report. Use this if you cannot attend but the cost should still be charged.

Only the booking owner, the resource's calendar admin, or a global admin can delete or release a booking. The tolerance and calendar admin are configured per resource in the Admin panel (⚙ Admin → Resources).

Grant assignment & financial report

When creating a booking you can assign it to a grant — only grants where your username is listed as a member are shown. Approved and Released bookings on resources with a pricing SKU appear automatically in the Dashboard expense table. The cost is calculated as:

cost = duration (hours) × SKU rate (EUR/h, by tier)

Full-day bookings count as 8 hours. The tier is taken from the grant if one is assigned, otherwise from the user's pricing tier. Released bookings are shown with an amber released badge in the report.

Service Requests

The Requests page (requests.html) is where users submit requests for core facility services such as genomics runs, cell sorting sessions, or other custom workflows. Admins review and action submissions.

Submitting a request

  1. Click New request
  2. Select the request type — Genomics, Cell Sorting, or Other
  3. Fill in the type-specific form fields (configured by admins per type)
  4. Click Submit

Your request appears in the list with Pending status. You will receive an email notification when it is reviewed.

Review & approval

Admins (or users with the configured approver role for the request type) see all pending requests in the same list. Clicking a request opens a detail view showing every submitted field.

The reviewer can Approve or Reject from the detail view, optionally adding a review note. An email is sent automatically to the requester's address with the decision.

Automatic Kanban project creation

When a request is approved, the system automatically creates a Kanban project for the requester. It contains one To Do task with all form metadata pre-filled in the notes:

  • All form fields as Key: Value lines
  • Submitter username
  • Approver username and approval date
ℹ

The Kanban project is owned by the requester. It appears immediately in their Mine scope on the board.

Grants

The Grants page (grants.html) is admin-only. It manages the funding grants used across the platform for financial tracking. Access requires the Admin role.

Grant fields

Field Description
Code Short identifier shown in dropdowns (e.g. GR-2025-01)
Name Full descriptive name of the grant or project
Tier Funding category: Internal, CRACs, Academia, or Industry
Funds Total allocated budget in EUR
Members Platform users who can assign this grant to their bookings and store orders

How grants are used

  • Bookings — grant members can select the grant when creating a booking; its cost is attributed to the grant in the expense table
  • Store orders — grant members can assign a grant code to individual SKU items in Kanban task cards
  • Dashboard CSV export — costs are broken down by grant code for financial reporting

Tips & Shortcuts

@

@mention teammates in notes

Type @username anywhere in a task note. After you stop typing for 30 seconds, the mentioned user receives an email notification pointing them to the project. Useful for flagging blockers or requesting input.

⌨

Esc cancels editing

Press Esc while editing a note or title to discard changes and return to the read view with no save. Clicking outside the editor saves changes.

🖰

Click a note to edit

Notes display as rendered markdown by default. Click anywhere on the note text to enter the markdown editor with the full formatting toolbar. Press Esc or click outside to exit.

⏱

Sessions expire after 8 hours

If you are redirected to the login page with an expired message, your session has timed out. Log back in to resume. Unsaved note edits will be lost — save frequently during long editing sessions.

💊

Automatic weekly report

Every Monday at 08:00 UTC the system generates an AI-written summary of blocked tasks and last week's activity. The report is emailed automatically to all team members who have an email address on file.

🔑

Changing your password

Click 🔑 Password in the top navigation bar to open the password change page. Passwords must be at least 10 characters and include an uppercase letter, a lowercase letter, a number, and a special character.

🚫

Rate limits

The API enforces 5 requests/minute on the login endpoint and 60 requests/minute elsewhere. If you receive a 429 Too Many Requests error, wait a moment before retrying.
